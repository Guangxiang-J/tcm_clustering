# TCM Constitution Clustering Analysis

This repository contains the reproducible analysis code for the manuscript **Unsupervised Clustering of Concurrent Constitutions in Traditional Chinese Medicine for Personalized Treatment**.

## 1. Project structure

```text
.
├── README.md
├── pyproject.toml
├── requirements.txt
├── scripts/
│   ├── run_main_analysis.py
│   ├── run_main_stability.py
│   └── run_threshold_sensitivity.py
└── src/
    └── tcm_clustering/
        ├── __init__.py
        ├── common.py
        ├── main_analysis.py
        ├── main_stability.py
        └── threshold_sensitivity.py
```

## 2. Software requirements

The analysis was designed for Python 3.10 or later. A clean virtual environment is recommended.

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
pip install -e .
```

## 3. Input data

The scripts accept `.xlsx`, `.xls`, or `.csv` input files.

By default, the following constitution-domain columns are excluded from clustering and treated as trait columns:

```text
YiDC YaDC QDC PDC DHC BSC SDC QSC
```

All remaining columns are treated as symptom variables. The main binary-encoded analysis should use the binary symptom dataset. The threshold sensitivity analysis should use the original 5-point Likert-scale symptom dataset.

Expected example input paths:

```text
data/clean_8_binary.xlsx
data/clean_8_COPY.xlsx
```

The raw questionnaire dataset is not included in the public repository because it contains participant-level data. Users who obtain access to the dataset should place the files in the `data/` directory and use the same file names or update the command-line arguments accordingly.

## 4. Step-by-step analysis

### Step 1. Run the main clustering analysis

This step compares dimensionality reduction methods and clustering algorithms, including PCA, t-SNE, UMAP, K-means, DBSCAN, and GMM.

```bash
python scripts/run_main_analysis.py   --input data/clean_8_binary.xlsx   --output results/main_analysis
```

Expected outputs include clustering metrics, selected parameters, cluster labels, top symptoms, and visualization files in `results/main_analysis/`.

### Step 2. Run repeated-run stability analysis

This step evaluates repeated-run stability for the selected clustering pipelines and summarizes model-selection stability and agreement across runs.

```bash
python scripts/run_main_stability.py   --input data/clean_8_binary.xlsx   --output results/main_stability_representative
```

Expected outputs include repeated-run metrics, selected K values, agreement statistics such as ARI and NMI, and representative-seed outputs in `results/main_stability_representative/`.

### Step 3. Run threshold sensitivity analysis

This step repeats the binary-encoding and clustering procedures under alternative Likert cutoffs.

```bash
python scripts/run_threshold_sensitivity.py   --input data/clean_8_COPY.xlsx   --output results/threshold_sensitivity   --thresholds 2 3 4
```

Expected outputs include threshold-specific metrics, stability summaries, selected K values, and cluster-size comparisons in `results/threshold_sensitivity/`.

## 5. Reproducibility notes

The scripts use command-line arguments rather than manual top-of-file parameter editing. Default settings preserve the analytical logic used in the manuscript, including K-search ranges, UMAP settings, K-means and GMM initialization rules, repeated-seed settings, and threshold-sensitivity procedures.

Each run writes output files and a run summary to the specified output directory. Keeping the `results/` directory generated by each step allows the tables and figures in the manuscript and supplementary materials to be reproduced.

## 6. Suggested execution order

```bash
python scripts/run_main_analysis.py --input data/clean_8_binary.xlsx --output results/main_analysis
python scripts/run_main_stability.py --input data/clean_8_binary.xlsx --output results/main_stability_representative
python scripts/run_threshold_sensitivity.py --input data/clean_8_COPY.xlsx --output results/threshold_sensitivity --thresholds 2 3 4
```
